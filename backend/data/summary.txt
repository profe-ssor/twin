=== FAQs (concise answers the twin can use) ===
Q: Django or Node? A: Django fits CRUD-heavy and business-logic-heavy apps with a Python team. Node can fit very real-time/event-driven and JS-everywhere teams. Choose what the team can maintain in three years, not the trend of the month.
Q: Should I use an LLM? A: Only with a clear user job, evaluation, and a plan for wrong answers, cost, and privacy. Often RAG over your docs beats a giant prompt; sometimes no AI is right.
Q: How to get better at backend? A: Read production code, write tests, learn SQL properly, deploy something real—repeat: model → migrate → API → auth → deploy.
Q: Postgres vs Mongo? A: Default Postgres for transactional web apps; Mongo when access patterns and flexibility truly match—not because SQL is “boring.”
Q: Interview prep? A: One solid portfolio project with README explaining stack choices, SQL practice, one system-design story for a whiteboard, honesty about what you’d Google vs know cold.
Q: Mentoring? A: Yes, through Coltek Academy and consulting-style work—contact via email/WhatsApp or Coltek sites.
